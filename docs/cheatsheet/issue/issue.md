チケット用テンプレート
====================

チケット用のテンプレートです

要望
--------------------

機能要望のテンプレートです

```md
概要
---------------------

<!-- 要望の概要を記載します -->

### 参考URL

<!-- 参考URLがあれば記載します -->

### 背景

<!-- なぜそのように思ったのかの理由や背景を記載します -->

内容(要件)
---------------------

<!--
どのようになればいいのかを箇条書きで記載します
(複数の要件がある場合はチケットを分けましょう)
-->

### 制約

<!-- 要件に対する制約を記載します -->

補足事項
---------------------

<!-- 補足事項などがあれば記載します -->
```

バグレポート
--------------------

バグレポートのテンプレートです

```md
概要
-------------------

<!-- どのような不具合が起こったのかを概要を記載します -->

### 事象が発生したURL

<!-- 事象が発生したURLを記載します -->

期待した結果
-------------------

<!-- 想定していた振る舞いを記載します -->

実際の結果
-------------------

<!--
画面に表示されたエラーログを記載したり画面キャプチャなどを添付し、
具体的にどのような不具合が発生したかわかる内容を記載します
-->

再現方法
-------------------

<!--
「事象が発生したURL」をクリックしただけ事象が確認できない場合は、
再現方法の操作方法等の再現手順を記載します。
-->

利用環境
-------------------

<!--
利用環境を記載します。
※バージョン情報も合わせて記載している方が望ましいです
※利用環境がわからない場合は「[ご利用環境チェックツール][]」などを利用することで調べることができます。

[ご利用環境チェックツール]: http://mysys-check.com/
-->

### デバイス

* PC(パソコン)
* スマートフォン
* タブレット

### OS/OSバージョン

* Windows
* Mac
* Linux
* Android
* iOS

### ブラウザ/ブラウザバージョン

* Chrome
* Edge
* Safari
* Firefox

原因
-------------------

<!-- 事象が発生した原因を(ここは調査した人が)記載します -->

対策
-------------------

<!-- どのように対応したのかを(ここは対応した人が)記載します -->
```

タスク
--------------------

タスクのテンプレートです

```md
概要
---------------------

<!-- 実装してほしい内容の概要を記載します -->

背景
---------------------

<!-- なぜその機能がほしいのかの理由や背景を記載します -->

内容(要件)
---------------------

<!-- 求める要件を記載します -->

### 制約

<!-- 要件に対する制約があれば記載します -->

詳細(概要設計)
---------------------

<!-- 概要設計の内容を(実装担当者が)記載します -->

制限事項
---------------------

<!-- 制限事項があれば記載します -->

備考
---------------------

<!-- 補足事項などがあれば記載します -->

テストケース
---------------------

<!-- 実装した機能のテスト内容を箇条書きで記載します -->
```
